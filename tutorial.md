# L2D Config Editor 新手使用教程

本教程面向第一次使用本工具的用户，按**推荐操作顺序**说明界面与常用功能。

---

## 一、这个工具是做什么的？

这是一个 **Live2D 相关 JSON 配置** 的可视化编辑器：用**节点**和**连线**在画布上搭出逻辑关系，在右侧填写参数，最后把结果保存成 **JSON 文件**。  
你可以把它理解成「在一张大图上摆积木，积木之间用线连起来」。

---

## 二、启动后第一件事：确认「工程目录」

1. 看窗口**最左侧**竖栏顶部的 **「配置文件」** 区域。
2.  点击 **「选择目录…」**，选中SVN文件夹。
3. 路径会显示在输入框里；若刚拷了文件进来，可点 **「刷新」**。

---

## 三、推荐操作顺序
| 1 | **新建**一个配置，或从列表里**双击**打开已有 JSO
| 2 | 画布上会有一个 **「初始节点」**；按界面要求把**最基础的信息**填完整（未完成前，往往不能随意加别的节点，状态栏会提示原因）
| 3 | 从初始节点的引脚出拉出线段，松手创建新节点（右键菜单也可以直接创建）。过渡动画用touchIdle节点，不设计idle切换的动画用touchDrag节点，特殊交互用备注节点，把需求写在里面。
| 4 | 用节点上的 **连线点** 把节点连起来（从输出拖到另一节点的输入），就像你在在线表格里写的那样。
| 6 | 需要改参数时，**单击选中节点**，改字段
| 7 | 改完后 **保存**，并提交到SVN

如果你打开的是**已有文件**，通常从第 3 步检查初始信息是否齐全即可，然后直接编辑、连线、保存。

## 四、界面分区说明

窗口大致是 **左 — 中 — 右** 三块，顶上是菜单栏。

### 4.1 左侧：配置文件

- **工程路径 +「选择目录…」**：决定当前列表从哪个文件夹扫 JSON（含子文件夹）。
- **刷新**：重新扫描目录。
- **新建**：新建一份配置（具体保存文件名会在你第一次保存时确定）。
- **重命名 / 删除**：针对列表里**当前选中的**文件（删除前请确认没有重要内容）。
- **文件列表**：
  - 带 **「[目录]」** 的行是**分组标题**（对应子文件夹），可点击**展开 / 收起**该组下的文件。
  - 下面的 **「配置 · …」** 是具体 JSON；**单击**选中，**双击**打开。

### 4.2 中间：搜索 + 画布

- **搜索**：输入关键字可匹配当前文档里节点上的内容；下方会列出结果，**点击结果**可跳到对应节点。
- **还原布局**：把画布的缩放、平移恢复到该文件里记录的状态（若你缩放乱了可以用）。
- **画布**：
  - 每个**彩色方块**是一个**节点**。
  - 方块之间有**曲线**，表示**连接关系**。
  - 右下角一般有简短提示：**中键拖动平移、滚轮缩放、Delete 删除** 等。

### 4.3 右侧：编辑模式、工具、Inspector

- **编辑模式 · 简易 / 高级**：同一节点在不同模式下**显示的字段多少不同**。新手可先用 **简易**，需要更多项时再切 **高级**（与菜单 **View → 编辑模式** 联动）。
- **工具**：
  - **优化布局**：自动整理**已经连上线**的节点位置，**不会**改你的字段内容和连线关系。
  - **节点垃圾箱**：查看已删除节点占用的编号等信息（与菜单 **View → 节点垃圾箱** 相同）。
- **Inspector**：**选中一个节点**后，这里出现该节点的表单；**没选中**时会提示「请选择一个节点」。
- **校验摘要**：当配置有问题时，这里会集中提示（便于你逐个修正）。

## 五、顶栏菜单（File / Edit / View）

### File（文件）

- **打开**：从磁盘选一个 JSON（不限于当前工程目录）。
- **保存**：保存当前文件（Ctrl+S）。
- **重载字段配置**：重新读取编辑器自带的字段定义（一般改程序或换 schema 后才需要；日常可忽略）。

### Edit（编辑）

- **撤销 / 重做**（Ctrl+Z / Ctrl+Y 或对应 redo 快捷键）。
- **复制 / 粘贴**：复制选中的节点（若该类型允许复制）。
- **复制节点**（Ctrl+D）：快速复制选中节点。
- **删除**：删除选中内容（与 Delete 键类似，具体以当前选中的是节点还是连线为准）。

### View（视图）

- **搜索**（Ctrl+F）：焦点到搜索框。
- **优化连线布局**：与右侧「优化布局」相同。
- **节点垃圾箱**：与右侧按钮相同。
- **CSV 预览**：用表格形式**预览**当前文档导出后的效果（便于检查行数和大致内容；列含义与生成规则本教程不展开）。
- **调试模式：显示 JSON 字段名**：开启后界面上会显示更偏技术的字段名，**新手建议保持关闭**。
- **编辑模式 → 简易模式 / 高级模式**：与右侧单选按钮一致。

## 六、画布上的常用操作

1. **平移**：按住**鼠标中键**拖动（或按画布角落提示的方式）。
2. **缩放**：**滚轮**。
3. **选中节点**：左键单击节点。
4. **多选**（若支持）：配合 Ctrl/Shift（以你本机实际操作为准，一般与常见软件类似）。
5. **移动节点**：拖动节点标题栏区域（不要误拖连线点）。
6. **添加节点**：在画布**空白处** **右键**，选 **「添加 TouchIdle」** 等（**初始节点**不会出现在这里，每个文档只有一个）。
7. **连线**：
   - 从一个节点的 **输出连线点** 按住拖到另一个节点的 **输入连线点** 上松开，即建立连接。
   - 若拖到**空白处**松开，可能会弹出 **「创建并连接 …」** 菜单，用于**顺便新建**一个节点并自动连上（省一步操作）。
8. **取消正在拉的线**：按 **Esc**。
9. **删除**：选中节点或连线后按 **Delete**（注意删除节点可能进「垃圾箱」逻辑，见工具说明）。

## 七、节点类型与含义（概念级）

下表帮助你看懂「这一坨彩色方块分别代表什么」。**不要求**你记住内部技术名，只要知道**角色**即可。

| 界面上的类型 | 大致含义 |
|--------------|----------|
| **初始节点** | 整份配置的**入口信息**：角色名、作者、ID、资源名等**全局相关**内容通常在这里。每个文件**只有一个**，且一般需要**先把它填到可用状态**，再添加其它功能节点。 |
| **TouchIdle** | 与 **待机 / 点击区域** 一类交互相关的功能节点（例如触摸 idle 状态）。可有多份，彼此用连线表达先后或分支关系。 |
| **TouchDrag** | 与 **拖拽** 一类交互相关的功能节点。同样可有多份，用连线组织流程。 |
| **备注节点** | **给你自己看的说明文字**，不参与导出逻辑；块可以**拉大**，适合写长说明。 |

同一类型可以有很多个节点，靠**位置、连线、右侧字段**区分它们各自的作用。

---

## 八、简易模式与高级模式

- **简易模式**：隐藏一部分不常用或偏细节的字段，界面更干净。
- **高级模式**：展开更多字段，适合已经熟悉流程、需要细调的用户。

两种模式改的是**同一份数据**，只是**显示层次**不同；随时切换不会丢内容。

---

## 九、保存与未保存提示

- 使用 **File → 保存** 或 **Ctrl+S** 把当前 JSON 写到磁盘。
- 若你**切换左侧另一个文件**、**换工程目录**或**关闭窗口**，而当前还有未保存修改，软件会按提示问你是否保存；**按自己需要选「保存 / 不保存 / 取消」**即可。
- 部分情况下软件也会在合适时机**自动保存**（以状态栏短暂提示为准），但**不要完全依赖自动保存**，重要修改请养成手动 Ctrl+S 的习惯。

## 十、新手易错点小结

1. **没选工程目录** → 左侧列表是空的或不对：先 **「选择目录…」** 再 **刷新**。
2. **加不了节点 / 提示要先完成初始信息** → 先选中 **初始节点**，在右侧把必填项填好。
3. **连线连不上** → 确认一端是 **输出**、另一端是 **输入**，且拖到**节点上的接点**而不是空白。
4. **找不到某个 JSON** → 它可能在**子文件夹**里，看分组标题是否折叠；或点 **刷新**。
5. **以为设置写进了项目里的 config.json** → 工程目录记忆在**系统设置**里，不是仓库里的那个 json。
