### 1. 工具概述与目标
设计一个基于 Python 的可视化类蓝图编辑器（类似 Unreal Blueprint 或节点式 Shader 编辑器），用于编辑 L2D 角色交互配置。  
编辑器输出 **JSON 文件**，后续由独立转换工具将 JSON 转换为符合规范的 **CSV 文件**（格式参照附件 `ship_l2d.csv`）。

---

### 2. 界面布局
- **Canvas 画布**：
  - 左上角带按键提示栏（可折叠/拖动）。
  - **支持鼠标滚轮缩放画布，按住鼠标中键拖拽平移画布。**
  - 支持 `Ctrl+Z` / `Ctrl+Y` 撤销重做。
- **左侧面板**：**文件浏览器**。
  - **列出当前工作目录下所有 `.json` 配置文件，点击可直接切换编辑。**
  - 支持刷新列表、新建配置文件。
- **右侧 Inspector**：属性编辑区。
  - 选中节点后，可在画布节点上直接编辑字段，也可在 Inspector 中编辑，两者实时同步。
- **底部或侧边**：**搜索栏**。
  - **支持全局搜索 JSON 字段值（例如搜索 `idle=13`），结果列表显示匹配的节点，点击可定位并高亮该节点。**
- **禁止使用 Emoji**。

---

### 3. 交互行为
| 操作 | 行为 |
|:---|:---|
| 从左侧文件浏览器打开 JSON 文件 | 加载配置并恢复画布节点布局与连线。 |
| 从节点库拖拽至画布松手 | 自动弹出右键菜单，可选择节点具体类型并立即创建于松手位置。 |
| `Ctrl+D` | 复制当前选中节点（包括属性值），新节点位置偏移放置，ID 按规则重新生成。 |
| `Ctrl+C/V` | 复制/粘贴选中节点（支持跨画布）。 |
| `Del` | 删除选中节点或连线。**初始节点不可删除**。 |
| 连线 | 每个节点有 1 个输入引脚和 1 个输出引脚，允许一对多、多对一连接。连线仅为编辑器内视觉辅助，**不参与最终 CSV 导出**。 |
| 撤销/重做 | `Ctrl+Z` / `Ctrl+Y`，支持节点增删、属性修改、连线变更的回退。 |
| 节点高亮警告 | 符合警告条件时，节点外框变为红色，问题解决后自动恢复。 |
| 保存 | `Ctrl+S` 导出为 JSON 文件（包含布局元数据），同时可触发 CSV 转换预览。 |

### 4. 节点类定义

#### 4.1 通用字段结构
每个节点由以下部分组成：
- `前提条件`（画布内可视的输入引脚，不导出数据）
- `子字段列表`：每个子字段包含：
  - 显示名称（画布上可见）
  - 内部键名（用于 JSON 导出）
  - 编辑模式：简易模式 / 高级模式（切换开关位于节点右上角）
  - 控件类型：输入框 / 下拉框 / 单选框 / 复选框 / 数字微调
  - 默认值
  - 可见性依赖规则

#### 4.2 初始节点（Initial Node）
**作用**：定义当前配置文件的元信息，每个画布**必须且只能存在一个**初始节点。  
**导出方式**：初始节点不直接生成 CSV 行，但其字段值会作为全局元数据写入 JSON 根对象，供转换工具使用。

| 显示名称 | 内部键名 | 控件类型 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|
| 备注 | `tips` | 单行文本框 | 空 | 内部标注，不导出到 CSV |
| 版本 | `version` | 日期选择器 | **2099-09-09** | **默认值为一个未来日期，提醒用户必须修改。** 格式 `YYYY-MM-DD` |
| 默认状态 | `defaultState` | 文本输入框 | `idle0` | 填写初始待机序号 |
| 角色名 | `CharName` | 单行文本框 | 空 | |
| 角色资源名 | `memo` | 单行文本框 | 空 | 对应 CSV 中的 `memo` 列 |
| 角色ID | `ship_skin_id` | 整数输入框 | 0 | 对应 CSV 中的 `ship_skin_id` 列 |
| 允许目光拖拽的待机 | `react_condition` | 整数输入框 | 0 | 对应 CSV 中的 `react_condition` 列 |
| 作者 | `author` | 单行文本框 | 空 | 内部使用，不导出 CSV |

#### 4.3 TouchIdle 节点
**作用**：定义触摸触发的待机动作切换。  
**CSV 对应行类型**：生成 `action_trigger` 和 `action_trigger_active` 字段。

##### 简易模式字段
| 显示名称 | 内部键名 | 控件类型 | 默认值 | 依赖规则 / 说明 |
|:---|:---|:---|:---|:---|
| 备注 | `tips` | 文本框 | 空 | |
| 框 | `draw_able_name` | 文本框 | 空 | 对应 CSV 列 `draw_able_name` |
| 参数 | `parameter` | 文本框 | `empty` | 对应 CSV 列 `parameter` |
| 参数范围 | `range` | 范围输入 `{min,max}` | `{0,1}` | 对应 CSV 列 `range` |
| 交互类型 | `control_type` | 下拉：点击/拖拽 | 点击 | 选择拖拽时出现下方字段，**并自动设置高级模式中 `drag_direct=1`（点击则为0）** |
| 拖拽方向 | `drag_ui_direction` | 下拉：上/下/左/右 | 无 | 仅当 `control_type=拖拽` 可见。**选择后自动映射到 `offset_x` 和 `offset_y` 字段，见第8条** |
| 过渡形式 | `transition_type` | 下拉：动画/硬切 | 动画 | 选择动画时出现下方字段 |
| 目标idle | **`target_idle`** | **整数输入框** | 1 | **用户输入的数值将用于自动生成 `action_trigger` 和 `action_trigger_active` 字段内容。** |
| 自动生成区 | （不可编辑） | 只读文本 | - | 实时预览生成的 `action_trigger` 和 `action_trigger_active` 内容。 |

##### 高级模式字段
高级模式提供与 CSV 列一一对应的细粒度控制。**简易模式中的 `target_idle` 不直接作为独立字段导出，而是用于生成两个字符串字段。**

| 显示名称 | 内部键名 | 控件类型 | 默认值 | CSV 列映射 |
|:---|:---|:---|:---|:---|
| ID | `id` | 自动生成 | `初始节点id+序号` | `id` 列 |
| 数值模式 | `mode` | 下拉：覆盖/加法/乘法 | 1 (覆盖) | `mode` 列 |
| 初始值 | `start_value` | 整数 | 0 | `start_value` 列 |
| 参数范围 | `range` | 范围输入 | `{0,1}` | `range` 列 |
| 数值吸附范围 | `parts_data` | 范围输入 | 空 | `parts_data` 列 |
| 屏蔽目光跟随 | `ignore_react` | 复选框 | 1 | `ignore_react` 列 |
| 非待机状态不可拖动 | `ignore_action` | 复选框 | 1 | `ignore_action` 列 |
| 使用绝对值 | `range_abs` | 复选框 | 当 range 含负值时强制为 0 | `range_abs` 列 |
| 方向响应 | `drag_direct` | 下拉：0不适用/1正向/2负向 | **根据简易模式 `control_type` 自动设1或0，高级模式可手动修改** | `drag_direct` 列 |
| 眼神跟随X | `react_pos_x` | 整数 | 空 | `react_pos_x` 列 |
| 眼神跟随Y | `react_pos_y` | 整数 | 空 | `react_pos_y` 列 |
| 左右移动影响拖拽 | `offset_x` | 整数 | 0 | `offset_x` 列 |
| 上下移动影响拖拽 | `offset_y` | 整数 | 0 | `offset_y` 列 |
| 开始拖拽响应系数 | `smooth` | 整数 | 100 | `smooth` 列 |
| 回弹拖拽响应系数 | `revert_smooth` | 整数 | 100 | `revert_smooth` 列 |
| 回弹延迟 | `revert` | 整数 | -1 | `revert` 列 |
| 陀螺仪启用 | `gyro` | 复选框 | 0 | `gyro` 列 |
| 陀螺仪X轴 | `gyro_x` | 整数 | 0 | `gyro_x` 列 |
| 陀螺仪Y轴 | `gyro_y` | 整数 | 0 | `gyro_y` 列 |
| 陀螺仪Z轴 | `gyro_z` | 整数 | 0 | `gyro_z` 列 |
| 触发间隔 | `limit_time` | 整数 | 1 | `limit_time` 列 |
| **动作触发器** | **`action_trigger`** | **文本框（自动生成可编辑）** | **自动生成** | **`action_trigger` 列** |
| **激活后操作** | **`action_trigger_active`** | **文本框（自动生成可编辑）** | **自动生成** | **`action_trigger_active` 列** |

#### 4.4 TouchDrag 节点
**作用**：定义拖拽交互及数值变化。  
**CSV 对应行类型**：生成包含 `action = 'touch_dragX'` 或纯参数控制行。

##### 简易模式字段
| 显示名称 | 内部键名 | 控件类型 | 默认值 | 依赖规则 |
|:---|:---|:---|:---|:---|
| 备注 | `tips` | 文本框 | 空 | |
| 框 | `draw_able_name` | 文本框 | `drag` | |
| 参数 | `parameter` | 文本框 | `touchdragX` | |
| 参数范围 | `range` | 范围输入 | `{0,1}` | |
| 交互类型 | `control_type` | 下拉：点击/拖拽 | 点击 | |
| 拖拽方向 | `drag_ui_direction` | 下拉：上/下/左/右 | 无 | 映射规则同 TouchIdle 节点 |
| 交互结果 | `result_type` | 下拉：播放动画/数值变化 | 播放动画 | |
| 目标idle | **`target_idle`** | **整数输入框** | 1 | 若选择“播放动画”，则自动生成 `action_trigger` 和 `action_trigger_active`（同 TouchIdle 规则） |
| 数值变化到 | `target_value` | 浮点数 | 1 | 仅当结果=数值变化时可见 |
| 数值是否回弹 | `revert_enabled` | 复选框 | 否 | 仅当结果=数值变化时可见 |
| **数值吸附** | **`parts_data`** | **文本框（逗号分隔数字）** | 空 | **格式如 `0,1,2`，非法输入时节点变红警告。对应 CSV 的 `parts_data` 列。** |

##### 高级模式字段
与 TouchIdle 高级模式基本一致，**同样包含 `action_trigger` 和 `action_trigger_active` 两个可编辑文本框**，生成逻辑见第6节。

#### 4.5 备注节点（Comment Node）
- 可自由调整尺寸，背景色半透明，层级低于所有功能节点。
- 拥有输入/输出引脚，**仅用于视觉连线，不参与任何数据导出或校验**。
- 包含一个多行文本字段 `content` 用于输入备注内容。

### 5. 关键字段自动生成规则

#### 5.1 `action_trigger` 生成规则（TouchIdle / TouchDrag）
- **简易模式显示值**：当用户在简易模式输入 `目标idle = N` 后，界面上显示预览文本 `touch_idleN`。
- **导出到 JSON / CSV 的格式**：`{type = 2 ,action = 'touch_idleN'}`（若为 Drag 且选择了交互动画，则 `action` 字段为用户填写的动画名）。
- **高级模式**：用户可直接编辑完整的 JSON 字符串，但建议保持与目标 idle 的同步。

#### 5.2 `action_trigger_active` 生成规则
- **当 `目标idle ≠ 0` 时**，自动生成模板：
  ```
  {enable = {},ignore = {'main_1','main_2','main_3','main_4','mission','mission_complete','complete','login','home','mail','touch_body','touch_head'},idle = $1}
  ```
  其中 `$1` 替换为用户输入的 `目标idle` 数值。
- **当 `目标idle = 0` 时**，固定输出：
  ```
  {enable = {},ignore = {},idle = 0}
  ```
（可能存在其他类似的我没有提及的字段生成规则，开发时记得留好接口）
- 高级模式下用户可手动修改该字段内容，但编辑器**不自动校验**与目标 idle 的一致性（由用户自行负责）。

#### 5.3 拖拽方向映射规则
当用户在简易模式选择**拖拽方向**时，自动填充以下高级字段：
| UI 选项 | 自动设置 |
|:---|:---|
| 上 | `offset_y = 100` |
| 下 | `offset_y = -100` |
| 左 | `offset_x = -100` |
| 右 | `offset_x = 100` |

同时，**当 `control_type = 拖拽` 时，`drag_direct` 自动设为 `1`；否则为 `0`**。高级模式下用户可覆盖这些值。

### 6. 警告条件（节点变红）

以下情况触发警告，相关节点边框显示红色：

1. **框序号冲突**  
   两个或以上 `TouchIdle` 节点具有相同的 `draw_able_name`，但生成的 `action_trigger` 或 `action_trigger_active` 内容不同。

2. **参数名重复**  
   两个或以上节点（不限类型）使用了相同的 `parameter` 值。该参数名在最终 CSV 中会导致列冲突，必须唯一。

3. **多个初始节点**  
   画布中存在超过一个 `初始节点`。初始节点不可删除，若用户通过复制产生第二个，立即变红提示。

4. **数值吸附格式非法**  
   `parts_data` 字段内容不是合法的逗号分隔数字列表（如 `0,1,2`）时，所属节点变红。

5. **范围与数值吸附冲突**  
   若 `parts_data` 中的数值超出了 `range` 定义的范围，变红警告。

> **已移除的警告**：不再对“目标idle缺失”或“默认状态无效”进行校验。

### 7. 导出 JSON 结构规范

编辑器导出的 JSON 文件应遵循以下结构（包含画布布局信息以便恢复）：

```json
{
  "meta": {
    "version": "2099-09-09",
    "author": "xxx",
    "ship_skin_id": 302291,
    "memo": "mingji_2",
    "default_state": "idle0",
    "react_condition": 0
  },
  "nodes": [
    {
      "id": 30229101,
      "type": "TouchIdle",
      "draw_able_name": "TouchIdle1",
      "parameter": "empty",
      "mode": 1,
      "start_value": 0,
      "range": "{0,1}",
      "parts_data": "",
      "ignore_react": 1,
      "ignore_action": 1,
      "range_abs": 1,
      "drag_direct": 1,
      "react_pos_x": null,
      "react_pos_y": null,
      "offset_x": 0,
      "offset_y": 100,
      "smooth": 100,
      "revert_smooth": 100,
      "revert": -1,
      "gyro": 0,
      "gyro_x": 0,
      "gyro_y": 0,
      "gyro_z": 0,
      "limit_time": 1,
      "action_trigger": "{type = 2 ,action = 'touch_idle1'}",
      "action_trigger_active": "{enable = {},ignore = {'main_1','main_2','main_3','main_4','mission','mission_complete','complete','login','home','mail','touch_body','touch_head'},idle = 1}",
      "shop_action": 0,
      "ui_position": { "x": 120, "y": 80 }
    }
  ],
  "connections": [
    { "from": "node_uuid_1", "to": "node_uuid_2" }
  ],
  "canvas_view": { "scale": 1.0, "offset_x": 0, "offset_y": 0 }
}
```

**节点顺序**：`nodes` 数组中的顺序即为最终 CSV 中行的顺序。默认按 `id` 升序排列。

### 9. 文件操作与搜索功能

- **左侧文件浏览器**：
  - 显示当前工作目录下所有 `.json` 文件。
  - 双击文件名加载配置，画布自动恢复节点布局与连线。
  - 支持新建、删除、重命名文件。
- **保存**：`Ctrl+S` 将当前画布状态保存为 JSON（包含布局信息）。
- **搜索**：
  - 快捷键 `Ctrl+F` 打开搜索栏。
  - 输入字符串（如 `idle=13`）后，实时过滤出所有包含该字符串的节点（包括 `action_trigger`、`action_trigger_active`、`target_idle` 等字段）。
  - 搜索结果列表显示节点类型和匹配字段摘要，点击可居中画布并高亮节点。
